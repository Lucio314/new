a) La commande "git pull" récupère les modifications à partir d'un dépôt distant et les fusionne avec la branche locale actuelle. En revanche, "git fetch" récupère uniquement les modifications du dépôt distant sans les fusionner avec la branche locale. 

b) Pour annuler les modifications locales non validées dans Git, nous pouvons utiliser la commande  "git checkout -- <fichier>" pour supprimer les modifications dans le répertoire de travail. 

c) Pour supprimer une branche distante dans Git, vous pouvez utiliser la commande "git push <dépôt distant> --delete <nom de la branche>" .

d) Pour voir l'historique des modifications d'un fichier spécifique dans Git, vous pouvez utiliser la commande "git log <fichier>".
